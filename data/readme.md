# Data Directory

This directory contains all data used for the Gene Expression Analysis of Lung Adenocarcinoma pipeline. The data is organized into `raw` (source data) and `processed` (cleaned, transformed, and integrated data) subdirectories.

## Structure Overview

```
data/
├── raw/                  # Original, unmodified source data from external databases
│   ├── Cancer Gene Census (Labeled Data)
│   ├── cBioPortal (RNA Seq Data)
│   └── Gtex (normal samples)
├── processed/           # Cleaned, harmonized, and feature-engineered data generated by the pipeline
│   ├── *_raw.csv        # Initial processed versions of raw data
│   ├── *_processed.csv  # Further cleaned/QC'd data
│   ├── *_harmonized.csv # Batch-corrected/normalized expression matrices
│   ├── *_features.csv   # Feature matrices for machine learning
│   ├── *_labels.csv     # Corresponding labels/targets
│   ├── *_scaled.csv     # Standardized/scaled features
│   └── various metadata and annotation files
```

## Key Processed Data Files

### Core Expression Data

- **`tumor_expression_harmonized.csv`** & **`normal_expression_harmonized.csv`**: The final, batch-corrected expression matrices (genes × samples) ready for analysis. These are the primary inputs for differential expression and machine learning.
- **`tumor_metadata_processed.csv`** & **`normal_metadata_processed.csv`**: Cleaned clinical/sample metadata corresponding to the expression matrices.

### Feature Engineering Outputs

- **`integrated_features.csv`**: The combined feature matrix (samples × features) containing expression, network, and potentially other features.
- **`train_features.csv`**, **`val_features.csv`**: Training and validation splits of the integrated features.
- **`train_labels.csv`**, **`val_labels.csv`**: Corresponding binary labels (e.g., cancer vs. normal) for the train/validation sets.
- **`cv_fold_assignments.csv`**: Cross-validation fold assignments for each sample.

### Gene Annotation & Filtering

- **`gene_annotation_table.csv`**: Mapping of gene identifiers (e.g., Ensembl IDs) to gene symbols and other annotations.
- **`common_genes.csv`**: List of genes common between tumor (cBioPortal) and normal (GTEx) datasets after processing.
- **`cancer_genes_raw.csv`**: List of genes from the Cancer Gene Census.
- **`unmatched_cgc_genes.csv`**: Cancer Gene Census genes not found in the common gene list.
- **`cgc_retention_report.csv`**: Summary of how many CGC genes were retained in the final feature set.

### Quality Control & Feature Selection

- **`qc_summary.csv`**: Summary statistics of data quality checks.
- **`highly_correlated_features.csv`**: List of features removed due to high correlation.
- **`expression_features.csv`** & **`network_features.csv`**: Subsets of features before integration.

## Usage Notes

- **Analysis Start Points**: Most analyses should begin with the `*_harmonized.csv` and `*_processed.csv` files in the `processed` folder.
- **Machine Learning Pipeline**: Use the `train_features.csv`, `train_labels.csv`, etc., for model training.
- **Reproducibility**: All processed files are generated automatically by the pipeline scripts. Do not manually edit these files.

For details on raw data sources, see [README in `data/raw`](./raw/README.md).
For details on the preprocessing pipeline, see [README in `data/processed`](./processed/README.md).

# Raw Data Directory

This directory contains the original, unmodified source data downloaded from public repositories. These files serve as the **input** to the preprocessing pipeline.

## Sources and Files

### 1. Cancer Gene Census (Labeled Data)

**Source**: COSMIC (Catalogue of Somatic Mutations in Cancer) Cancer Gene Census

**File**: `Census_allWed Mar 18 05_48_31 2026.csv`

**Purpose**: Provides a curated list of genes with documented roles in cancer. Used as a gold-standard set for labeling and feature prioritization.

**Note**: The filename includes a timestamp indicating the download date.

### 2. cBioPortal (RNA Seq Data) – **Tumor Samples**

**Source**: cBioPortal for Cancer Genomics (TCGA Lung Adenocarcinoma - LUAD)

**Files**:

-`data_mrna_seq_v2_rsem.txt`: RNA-Seq expression data (log2-transformed RSEM values) for tumor samples.

-`data_clinical_patient.txt`: Clinical metadata (patient age, stage, survival, etc.) for the tumor cohort.

**Purpose**: Primary source of lung adenocarcinoma tumor gene expression and associated clinical data.

### 3. GTEx (normal samples) – **Normal/Control Samples**

**Source**: GTEx (Genotype-Tissue Expression) Consortium, V11 release

**Files**:

-`gene_tpm_v11_lung.gct.gz`: Gene expression data (TPM values) for normal lung tissue samples.

-`GTEx_Analysis_v11_Annotations_SampleAttributesDD.xlsx`: Sample-level metadata (tissue details, processing batch).

-`GTEx_Analysis_v11_Annotations_SubjectPhenotypesDD.xlsx`: Donor/subject-level metadata (demographics).

**Purpose**: Source of healthy lung tissue expression data for comparison/control.

## Important Notes

-**Do Not Modify**: These files should never be edited manually. They are the reproducible starting point.

-**Format**: Files are in their native formats (.txt, .csv, .xlsx, .gct.gz).

-**License & Citation**: When using results from this pipeline, ensure proper citation of COSMIC, cBioPortal/TCGA, and GTEx according to their data usage policies.

-**Storage**: The `gene_tpm_v11_lung.gct.gz` is compressed. The pipeline script decompresses it during processing.

## Pipeline Input

The preprocessing scripts (`scripts/preprocessing/`) are configured to read directly from these files and paths.

# Processed Data Directory

This directory contains all **output files** generated by the automated preprocessing and feature engineering pipeline. Files are staged by processing step.

## Processing Pipeline Stages & File Nomenclature

### Stage 1: Raw Processing (`*_raw.csv`)
Initial cleaning and conversion of raw data into analyzable tables.
- **`tumor_expression_raw.csv`**, **`normal_expression_raw.csv`**: Expression matrices after basic parsing and format conversion.
- **`tumor_metadata_raw.csv`**, **`normal_metadata_raw.csv`**: Metadata extracted and converted to clean tables.

### Stage 2: Quality Control & Filtering (`*_processed.csv`)
After sample/gene filtering, outlier removal, and missing value handling.
- **`tumor_expression_processed.csv`**, **`normal_expression_processed.csv`**: Filtered expression matrices.
- **`qc_summary.csv`**: Report of samples/genes removed and QC metrics.

### Stage 3: Harmonization (`*_harmonized.csv`)
After batch correction, normalization, and alignment between tumor and normal datasets.
- **`tumor_expression_harmonized.csv`**, **`normal_expression_harmonized.csv`**: **Final expression matrices**. Genes are aligned (common genes), and distributions are adjusted for comparative analysis.

### Stage 4: Feature Engineering (`*_features.csv`)
Creation of feature matrices for machine learning.
- **`expression_features.csv`**: Features derived directly from harmonized expression data.
- **`network_features.csv`**: Features derived from gene interaction networks (e.g., pathway activity scores).
- **`integrated_features.csv`**: Combined matrix of all features (expression + network).
- **`integrated_features_scaled.csv`**: Z-score standardized version of integrated features.

### Stage 5: Label Creation & Dataset Splitting (`*_labels.csv`, `*_assignments.csv`)
- **`gene_labels.csv`**: Binary labels for genes (e.g., "cancer gene" vs. "non-cancer gene") based on Cancer Gene Census.
- **`train_labels.csv`**, **`val_labels.csv`**: Sample-level labels for the train/validation splits.
- **`cv_fold_assignments.csv`**: Cross-validation fold assignments for rigorous evaluation.

### Stage 6: Annotation & Gene Sets
- **`gene_annotation_table.csv`**: Master gene annotation lookup table.
- **`common_genes.csv`**: Final list of genes used in analysis.
- **`cancer_genes_raw.csv`**: Original CGC list.
- **`unmatched_cgc_genes.csv`**: CGC genes not found in common genes.
- **`cgc_retention_report.csv`**: Summary of CGC gene retention.

## How to Use These Files

### For Differential Expression Analysis
Start with:
- `tumor_expression_harmonized.csv`
- `normal_expression_harmonized.csv`
Optional: `gene_annotation_table.csv` for gene symbol mapping.

### For Machine Learning Training
Use the ready-made splits:
- **Training**: `train_features.csv` (or `train_features_scaled.csv`) + `train_labels.csv`
- **Validation**: `val_features.csv` (or `val_features_scaled.csv`) + `val_labels.csv`
- **Cross-Validation**: Use `cv_fold_assignments.csv` with `integrated_features.csv` and `gene_labels.csv`.

### For Gene Set Analysis
Use:
- `common_genes.csv` as your universe.
- `cancer_genes_raw.csv` as your target set.
- `unmatched_cgc_genes.csv` to see what was lost.

## Reproducibility
All files in this directory are generated by the pipeline. To regenerate them, run the main preprocessing script. **Do not edit these files manually**; any changes should be made to the pipeline source code.
